# RAG README (Current App State)

This document reflects the current Retrieval-Augmented Generation implementation in this repository.

## Snapshot

- RAG retrieval is active in [app/api/chat/route.ts](app/api/chat/route.ts).
- PDF ingestion and chunk embedding are active in [app/upload/action.ts](app/upload/action.ts).
- Chat completions are sent through Groq-compatible OpenAI API in [app/api/chat/route.ts](app/api/chat/route.ts).
- Embeddings are generated through OpenRouter in [lib/embeddings.ts](lib/embeddings.ts).
- Retrieval defaults are `limit=5` and `threshold=0.25` in [lib/search.ts](lib/search.ts).
- A debug diagnostic endpoint exists at [app/api/debug/diagnostic/route.ts](app/api/debug/diagnostic/route.ts).

## End-to-End Flow

1. A user uploads a PDF from [app/upload/page.tsx](app/upload/page.tsx).
2. [app/upload/action.ts](app/upload/action.ts) parses text with `pdf-parse` (`PDFParse` class API).
3. Text is chunked using [lib/chunking.ts](lib/chunking.ts).
4. Chunk embeddings are generated by [lib/embeddings.ts](lib/embeddings.ts).
5. Chunk records are inserted into `documents` via [lib/db-schema.ts](lib/db-schema.ts).
6. Chat route extracts the latest user query in [app/api/chat/route.ts](app/api/chat/route.ts).
7. Query embedding search runs via [lib/search.ts](lib/search.ts).
8. Top matches are formatted as `[Document N]` context blocks and appended to the instruction prompt.
9. Response is streamed back to the UI using `streamText`.

## Models and Providers

### Chat generation

- Provider client: Groq-compatible endpoint in [app/api/chat/route.ts](app/api/chat/route.ts).
- Base URL: `https://api.groq.com/openai/v1`.
- Model source: `GROQ_CHAT_MODEL` env var.
- Current fallback default in code: `moonshotai/kimi-k2-instruct-0905`.

### Embeddings

- Provider client: OpenRouter in [lib/embeddings.ts](lib/embeddings.ts).
- Base URL: `https://openrouter.ai/api/v1`.
- Model: `nvidia/llama-nemotron-embed-vl-1b-v2:free`.
- Embedding dimension normalized to `2000` to match DB vector size.

## Retrieval Details

- Query function: `searchDocuments(query, limit = 5, threshold = 0.25)` in [lib/search.ts](lib/search.ts).
- Similarity metric: cosine similarity computed as `1 - cosineDistance(...)`.
- Filter: only results with similarity greater than threshold.
- Ordering: descending similarity.

## Database Notes

- Table: `documents`.
- Key columns: `id`, `content`, `emdeddings`.
- Vector column name is currently spelled `emdeddings` in schema and queries.

## Context Injection Behavior

- Route builds a base instruction prompt.
- Retrieved chunks are appended into prompt context as `[Document 1]`, `[Document 2]`, and so on.
- For models detected as Gemma variants, system instructions are inlined into the first user message.
- For other models, context is sent through the system channel.
- Automatic retry exists for system-channel compatibility errors.

## Diagnostics

- Endpoint: `GET /api/debug/diagnostic?query=...` and `POST /api/debug/diagnostic`.
- File: [app/api/debug/diagnostic/route.ts](app/api/debug/diagnostic/route.ts).
- Reports:
1. Total documents and embedding counts.
2. Top similarity scores.
3. Match counts for thresholds 0.5, 0.3, and 0.2.

## Required Environment Variables

- `NEON_DATABASE_URL`
- `GROQ_API_KEY`
- `GROQ_CHAT_MODEL` (optional override)
- `OPENROUTER_API_KEY` (required for embeddings)

Current code in [app/api/chat/route.ts](app/api/chat/route.ts) still checks `OPENROUTER_API_KEY` during chat requests. Since chat now uses Groq, this check should be aligned in code if you want provider checks to be strict and accurate.

## Quick Validation Checklist

1. Run `npx drizzle-kit push` successfully.
2. Upload a PDF and confirm `created X searchable chunks` response.
3. Call diagnostic endpoint and confirm non-zero `totalDocuments`.
4. Ask a query from uploaded text and confirm retrieval logs show non-zero matches.
5. Verify context appears in logs before model call.

## Current Gaps

- Chat UI does not render explicit source citations.
- Retrieval is global, not scoped per-user.
- There is no retrieval metrics dashboard.
